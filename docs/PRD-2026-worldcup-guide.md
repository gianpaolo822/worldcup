## 1. Summary（摘要）

**2026 美加墨世界杯观赛指南**是一款面向中文用户的、非官方移动端 Web 数据查询工具。用户在赛期需要快速查看赛程、积分榜、比赛详情（阵容/事件/数据）与历史资料，但主流 App 信息分散、广告多，且缺少针对 2026 扩军赛制（48 队 / 12 组）的清晰呈现。

本产品以**静态 JSON + Git 维护 + Vercel 部署**为架构，一期不做账号、推送与实时 API，优先验证「有人愿意用链接查世界杯数据」这一核心假设。截至 2026-06-04，用户端四 Tab 与比赛/球队/球员详情页已实现，约 46/104 场比赛具备完整阵容与事件数据，下一步重点是公开部署与小范围用户验证。

## 3. Background（背景）

### 3.1  initiative 是什么？

2026 年 FIFA 世界杯首次在北美三国举办，参赛队扩至 **48 支、12 个小组**，淘汰赛从 32 强起步。中文观众在赛期的高频需求包括：

- 今天/本周有哪些比赛？开球时间（北京时间）？
- 某组积分榜与第三晋级形势如何？
- 某场比赛谁进球、谁换人、阵容与数据统计？
- 某队名单、伤病/退赛、队内射手榜？
- 历届冠军与历史射手？

本项目将上述能力整合为**单一移动端 Web**，供个人学习与小范围分享使用。

### 3.2 为什么现在做？

| 变化 | 影响 |
|------|------|
| 2026 赛期临近 | 赛程与比分维护有明确时间窗口 |
| 48 队扩军 | 传统「8 组」产品难以直接套用，需自定义第三晋级规则 |
| AI 辅助开发成熟 | 单人可在数周内完成 MVP 级 Web + 数据流水线 |
| 开放数据可用 | openfootball、手工 JSON、公开赛报可支撑非实时场景 |

### 3.3 是否最近才变得可行？

是。在以下组合成立前，单人维护「赛期每日更新 + 多语言球员名 + 阵容图」成本过高：

1. **静态站点 + JSON bundle**（无需后端即可上线）  
2. **脚本化 merge / 积分榜计算**（改 `overrides.json` 即可更新比分）  
3. **Agent 辅助录入**（赛后从公开赛报写入 `match-lineups.json`）

## 4. Objective（目标）

### 4.1 产品目标

**让用户在一个链接里完成赛期 80% 的「查数据」需求**，无需安装 App、无需注册；维护者通过 Git + JSON 在 15 分钟内完成一场比赛的赛后更新。

### 4.2 对公司 / 用户的价值

| 对象 | 价值 |
|------|------|
| 用户 | 赛程筛选、中文球员名、阵容/事件/统计、扩军积分榜与淘汰赛 bracket |
| 维护者 | 可复用的「静态 JSON 世界杯产品」模板，学习数据流水线 + 前端一体化 |
| — | 非商业项目，无直接收入目标 |

### 4.3 与愿景的对齐

愿景：**轻量、合规、可维护的中文世界杯观赛参考站**。  
策略：**Web 优先验证 → 必要时二期 Supabase + 管理后台 + 小程序**。

## 5. Market Segment(s)（市场细分）

> 细分依据是**待完成的任务**，不是年龄或性别。

### 5.1 核心细分：赛期「快速查数据」观众

- **场景**：赛前看阵容、赛中/赛后看比分与事件、小组赛阶段看积分与出线形势  
- **痛点**：信息分散、广告多、扩军赛制理解成本高、英文球员名阅读慢  
- **当前替代方案**：微博/懂球帝/直播吧、FIFA 官网、搜索引擎  
- **约束**：移动端为主；可能从微信链接打开；接受 1–3 分钟数据延迟（静态部署）

### 5.2 次要细分：深度球迷 / 数据爱好者

- **场景**：看阵容图、换人时间线、队内射手榜、历史射手对比  
- **痛点**：免费工具很少同时提供「阵容 + 事件 + 中文名 + 扩军 bracket」  
- **约束**：可接受 UI 简洁，但数据准确性与可追溯性（来源）重要

### 5.3 维护者细分（内部用户）

- **场景**：赛后 30 分钟内更新比分、进球、黄牌、阵容  
- **痛点**：多源赛报格式不一；手动对齐球员 ID  
- **约束**：熟悉 Git；可运行 `npm run data:*`；无专职运营

## 7. Solution（方案）

### 7.1 UX / 信息架构

```
底部 Tab
├── 赛程 (/)           … 倒计时、Hero、日期区间、球队筛选、比赛列表
├── 积分榜 (/standings) … 小组赛 / 淘汰赛、12 组锚点、bracket
├── 数据榜 (/stats)     … 进球/助攻/黄红牌榜（来自 events）
└── 历史 (/history)     … 历届冠军、历史射手

详情（无 Tab）
├── /match/:id          … 比分、同组积分、阵容、事件、技术统计
├── /team/:id           … 赛程 / 阵容 / 概况（含伤退、队内榜）
└── /player/:id         … 资料、俱乐部、伤病、历史射手 bio
```

**关键用户流**

1. 打开链接 → 看到今日赛程 → 点击比赛 → 查看阵容与进球  
2. 积分榜 → 选择小组 → 点击球队 → 查看阵容与队内射手  
3. 数据榜 → 点击球员 → 跳转球员详情  

**设计原则**

- 移动优先（375px 基准），暗色主题 + 绿色 accent  
- 仅用 emoji 国旗，不用官方队徽（合规）  
- 详情页支持 `returnTo` 返回来源
